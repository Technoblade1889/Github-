# Github学习笔记



## 一、关于GitHub的初步理解 

我的理解简单说就是：**可以存代码、和别人一起写代码的网站，同时可以找到很多资源。**

## 二、初学的三个词

| 名词   | 个人理解                                                     |
| ------ | ------------------------------------------------------------ |
| 仓库   | 用来存项目的文件夹，一个项目对应一个仓库。                   |
| 分支   | 我理解为 项目的不同版本线，`main`是主版本。我写的时候会想着开分支，但现在还没很搞清楚。 |
| Commit | 每次修改代码后的保存操作。                                   |

## 三、最常用的操作

### 1. 新建仓库

1. 打开 GitHub 主页，点 + 号，点新仓库
2. 填仓库名
3. 选开源，许可证选MIT

### 2. 上传文件

打开库后直接拉readme文件进入就行，所以要学习markdown

### 3. 改文件

点仓库里的文件→点右上角铅笔图标（编辑）→改内容→拉到最下面点 “Commit changes”



## 四、踩过的坑

- 仓库名不能空格，用短横线`-`代替
- 第一次提交要写 Commit message
- 网页版目前够用，但复杂操作网课讲到得用 Git 软件，目前尚不清楚作用，只是下载在了桌面，后面再深入学习
